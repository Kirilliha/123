# Часть 1. Теория

## 1.1 Из чего состоит полносвязная сеть

```
вход X  →  [Linear → активация]  ×  n  →  Linear → выход
            \_______ скрытый слой _______/        \__ logits / probs __/
```

- **Входной слой** — это просто признаки `X` формы `(m, in_features)`, где `m` — батч.
- **Скрытые слои** — пары `Linear + нелинейность`. Именно нелинейность позволяет
  сети учить НЕ-линейные функции (без неё n линейных слоёв = один линейный).
- **Выходной слой** — Linear + соответствующая активация:
  - для бинарной классификации: Sigmoid,
  - для многоклассовой: Softmax.

## 1.2 Зачем веса и смещения

`Z = X·W + b` — это обучаемая линейная комбинация входов.

- `W` (веса) задаёт, **какие комбинации признаков** важны для каждого выходного
  нейрона. По сути — `out_features` разных "детекторов признаков".
- `b` (смещения) сдвигает порог активации. Без `b` все гиперплоскости проходят
  через ноль входа, что сильно ограничивает класс выучиваемых функций.

## 1.3 Forward pass и размерности

Для одного слоя:

```
Z = X · W + b
```

Пример: вход 4 признака, скрытый слой 5 нейронов, выход 3 класса, `batch=32`.

| Тензор | Размер | Комментарий |
|---|---|---|
| `X`       | `(32, 4)` | батч из 32 примеров |
| `W₁`      | `(4, 5)`  | веса первого Linear |
| `b₁`      | `(1, 5)`  | смещения, броадкастятся по строкам |
| `Z₁ = X·W₁ + b₁` | `(32, 5)` | до активации |
| `A₁ = ReLU(Z₁)` | `(32, 5)` | поэлементно |
| `W₂`      | `(5, 3)`  | веса второго Linear |
| `b₂`      | `(1, 3)`  | |
| `Z₂ = A₁·W₂ + b₂` | `(32, 3)` | логиты — вход в Softmax |
| `P = softmax(Z₂)` | `(32, 3)` | вероятности классов, по строке сумма = 1 |

## 1.4 Активации

### ReLU (скрытые слои)
```
ReLU(z) = max(0, z)
ReLU'(z) = 1, если z > 0;  0, если z ≤ 0
```

### Softmax (выход для классификации)
```
softmax(z)_k = exp(z_k) / Σ_j exp(z_j)
```

Численно стабильная версия — вычитаем максимум по строке перед `exp`,
иначе при больших `z` получим overflow:
```
z' = z - max(z)
softmax(z) = exp(z') / sum(exp(z'))
```

### Почему не Sigmoid в глубоких сетях?

```
sigmoid(z) = 1 / (1 + e^(-z))
sigmoid'(z) = s(z) * (1 - s(z)) ≤ 0.25
```

Производная всегда ≤ 0.25. При backprop градиент перемножается на производные
активаций каждого слоя. На 10 слоях: `0.25^10 ≈ 1e-6` — **vanishing gradient**,
сеть перестаёт обучать ранние слои. У ReLU производная = 1 в активной зоне,
поэтому градиент проходит без затухания.

## 1.5 Функция потерь: Cross-Entropy для классификации

Для одного примера с правильным классом `y`:
```
CE = -log(p_y),  где p — softmax-вероятности
```

Для батча — среднее:
```
L = -(1/m) · Σ_i log(p_{i, y_i})
```

### Почему не MSE?

1. **Градиент MSE** через softmax «мягкий», насыщающийся: `dL/dZ ∝ (p - y) * p * (1-p)`,
   и при сильно ошибочной уверенной модели (`p ≈ 0` для нужного класса) множитель
   `p*(1-p) ≈ 0` — градиент почти нулевой, сеть «зависает».
2. **CE-градиент** через softmax: `dL/dZ = p - y` — линейный по ошибке, никакого
   насыщения. Чем больше ошибка — тем больше шаг.
3. CE — это **максимизация правдоподобия** для категориального распределения,
   что в задачах классификации концептуально правильно.

## 1.6 Backpropagation через цепное правило

Допустим, есть блок `Linear → ReLU → Linear → SoftmaxCE`.

Цепное правило в матричной форме: если `L = f(g(W))`, то
```
∂L/∂W = (∂g/∂W)ᵀ · ∂L/∂g
```

### Шаг 1. Softmax + CE → dL/dZ выходного слоя
Аналитический вывод (один из самых красивых в ML) даёт:
```
dL/dZ_out = (P - Y_onehot) / m
```
где `m` — размер батча (потому что лосс — среднее по батчу).

### Шаг 2. Linear → dW, db, dX
Для `Z = X·W + b`:
```
dL/dW = Xᵀ · dL/dZ          shape: (in, m) · (m, out) = (in, out) ✓
dL/db = sum(dL/dZ, axis=0)  shape: (1, out) ✓
dL/dX = dL/dZ · Wᵀ          shape: (m, out) · (out, in) = (m, in)  → передаём дальше
```

**Соглашение в этом коде:** фактор `1/m` уже учтён в loss-функции
(она возвращает `(P - Y) / m`). Поэтому `Linear.backward` НЕ делит на `m` ещё
раз — иначе получится двойное усреднение, и градиент будет в `m` раз меньше
правильного (типичный баг, который ловит gradient check).

### Шаг 3. ReLU → dL/dX_prev
```
dL/dX_relu_in = dL/dY * 1[X > 0]
```
(производная ReLU — индикатор положительности).

### Шаг 4. Возвращаемся в предыдущий Linear, повторяем шаг 2

И так до самого входа.

### Почему `dW = X^T · dZ` (без батча)?

Для одного примера `z = x·W`, `dL/dW = x^T · (dL/dz)` — внешнее произведение
вектора входа и вектора градиента. Для батча просто складываем такие внешние
произведения по примерам, что и записывается как `X^T · dZ`. Деление на `m`
живёт в loss — мы хотим минимизировать **среднюю** ошибку, а не сумму.

### Контрольные вопросы по части 1

1. Почему в выражении `dL/dW = X^T · dZ` стоит именно `X^T`, а не `X`? Проверь
   через размерности: `X: (m, in)`, `dZ: (m, out)`, и мы хотим `dW: (in, out)`.
2. Если убрать ReLU между двумя Linear-слоями, какую функцию по `x` сможет

# Часть 2. Архитектура классов (`nn.py`)

Интерфейс каждого слоя:
```python
class Layer:
    def forward(self, x): ...
    def backward(self, grad_output): ...   # возвращает grad по входу
    def get_params(self): return []         # [W, b] для Linear, [] для активаций
    def get_grads(self): return []          # [dW, db] для Linear
    def update_params(self, lr): ...        # in-place: param -= lr * grad
```

### He-initialization

```python
W = np.random.randn(in_features, out_features) * sqrt(2 / in_features)
b = np.zeros((1, out_features))
```

- **Почему не нули для W?** Все нейроны слоя получали бы одинаковые градиенты,
  оставались бы симметричны, и слой эффективно был бы одним нейроном.
- **Почему именно He, а не просто `randn`?** Без масштабирования дисперсия
  активаций уплывает по слоям (растёт или затухает экспоненциально). He
  доказан как «правильный» масштаб для ReLU: сохраняет `Var(a) ≈ Var(x)`
  от слоя к слою. Для tanh/sigmoid используют Xavier (`sqrt(1/in)`).
- **`b` можно нулями** — асимметрию уже обеспечивает W, а нули для смещения
  никаких градиентов не «обнуляют».

### Softmax + CE в одном классе

Делаем `SoftmaxCrossEntropy` единым блоком ради:
- **простого градиента** `(P - Y)/m`;
- **численной стабильности**: отдельно Softmax → log в CE даёт `log(0)`-проблемы
  при `p_k → 0`. Совместная формула этих проблем не имеет.

# Часть 4. Обязательные проверки

## Shape-check
В `Linear.forward` и `Linear.backward` стоит `raise ValueError`, если входная
форма не та. Это ловит 80% ошибок «забыл reshape» при сборке сети.

## Gradient check (`grad_check.py`)

Идея: для случайных `x, y` посчитаем
- **аналитический** градиент через backward,
- **численный** градиент через `(L(w+ε) - L(w-ε)) / (2ε)` для каждой компоненты.

Относительная разница должна быть `< 1e-6` (для float64 обычно `~1e-10`).

Запусти:
```
$ python grad_check.py
  layer1.W    shape=(3, 5)        max rel err = 8.83e-10  [OK]
  layer1.b    shape=(1, 5)        max rel err = 3.12e-11  [OK]
  layer2.W    shape=(5, 4)        max rel err = 7.48e-10  [OK]
  layer2.b    shape=(1, 4)        max rel err = 3.16e-11  [OK]
```

Если хоть один FAIL — backward сломан, не идём дальше.

# Часть 6. Чек-лист отладки

### Loss не падает / колеблется
- Слишком большой `lr` — поделить на 10.
- Не вычитал max в softmax — могут быть `nan`-ы, забивающие градиент.
- Забыл занулять/обновить градиенты между шагами (в этом каркасе они
  перезаписываются в backward — но если будешь добавлять momentum/Adam, не
  забудь).
- Случайно перемешал индексы X и y (`y` оказался не от того примера).

### Loss = `nan` / `inf`
- В `log(p)` попал `p = 0` — нет ли клипа `eps`?
- `exp(z)` для огромного `z` без стабилизирующего вычитания max.
- Огромный `lr` → `W` улетели в бесконечность за один шаг.
- Деление на `p*(1-p)` в BCE при `p` ровно `0/1` — там должен быть `clip(p, eps, 1-eps)`.

### Train acc высокая, val низкая (переобучение)
- Сеть слишком большая для объёма данных. Уменьши слой / число слоёв.
- Добавь регуляризацию (L2-штраф к loss, Dropout — см. часть 7).
- Учишь слишком долго — останови по val_loss (early stopping).
- Аугментация данных (для табличных — добавить гауссов шум; для картинок —
  отдельная история).

### Все предсказания — один класс
- Несбалансированный датасет — модель «угадывает» большинство. Перевзвесь loss
  или переразбалансируй батч.
- `lr` слишком большой → веса схлопнулись.
- Лосс не подходит задаче (например, MSE поверх softmax вместо CE).
- Забыл вычесть max в softmax → `nan` → argmax возвращает первый по индексу.

### Vanishing gradients (градиенты близки к нулю)
- Sigmoid/tanh в глубоких слоях — заменить на ReLU.
- Плохая инициализация (нули или слишком маленькие веса). Использовать He.
- ReLU «мёртвая» — нейрон всё время отдаёт 0, градиент тоже 0 (dying ReLU).
  Лечится LeakyReLU/ELU или меньшим `lr`.

### `np.allclose(analytic, numeric)` падает
1. Проверь, что **сохраняешь копии** аналитических градиентов **до** второго
   forward — иначе numerical-цикл их перезаписывает.
2. Двойное деление на `m` — частая ошибка (одно в loss, второе в Linear).
3. Перепутал `X` и `X.T` в backward.
4. Берёшь `ε` слишком маленьким (<1e-7 для float64) → шум доминирует.
5. Локальная негладкость: производная ReLU в точке 0. Если `x` слишком близок
   к 0, отрешай или используй `x + 0.5`.

# Часть 7. Что дальше

Эти штуки сейчас НЕ нужны, но укладываются в тот же каркас:

### Dropout
Новый слой `Dropout(p)`:
- forward (train): создать маску `mask = (rand(shape) > p) / (1 - p)`,
  вернуть `x * mask`. Сохранить `mask`.
- forward (eval): вернуть `x` без изменений (поэтому в Layer нужно поле
  `self.training`).
- backward: вернуть `grad_output * mask`.
Помогает с переобучением: учим сеть не полагаться на конкретные нейроны.

### Batch / Layer Normalization
Новый слой, который для каждого батча (BN) или примера (LN) считает
`(x - mean) / sqrt(var + eps)` и умножает на обучаемые `γ`, добавляет `β`.
- В forward кэширует `mean, var, x_norm`.
- В backward — длинный, но прямой вывод цепным правилом.
- BN добавляет moving averages mean/var для inference.
- Резко ускоряет обучение глубоких сетей.

### Conv2D через im2col
Свёрточный слой:
- разворачиваем все патчи входа в столбцы матрицы `X_col` функцией `im2col`;
- свёртка превращается в `X_col @ W_col`;
- backward — через `col2im`.
Это позволяет переиспользовать тот же `Linear`-механизм. Минусы: память
`O(K²·H·W·C_in)` против обычной свёртки.

### SGD с momentum, Adam
Заменяем `update_params`. Для **momentum**: храним `v = μ·v - lr·dW`,
`W += v`. Для **Adam**: храним first/second moments `m`, `v` с
экспоненциальным сглаживанием, делаем bias-correction, шаг
`W -= lr · m̂ / (sqrt(v̂) + ε)`. Уменьшает зависимость от выбора `lr`.

### Сохранение/загрузка через pickle
```python
import pickle
def save(net, path):
    state = [(layer.W.copy(), layer.b.copy()) if isinstance(layer, Linear) else None
             for layer in net.layers]
    with open(path, 'wb') as f: pickle.dump(state, f)
def load(net, path):
    with open(path, 'rb') as f: state = pickle.load(f)
    for layer, st in zip(net.layers, state):
        if st is not None:
            layer.W[...], layer.b[...] = st  # in-place чтобы не порвать ссылки
```
